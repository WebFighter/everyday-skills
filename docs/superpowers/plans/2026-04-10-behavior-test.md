# 行为测试 Skill 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 创建纯行为驱动的趣味测试 skill，36 题测量 12 维度，输出 16 种行为类型归档 + 多维评分。

**Architecture:** 单文件 SKILL.md，包含前置信息、36 道场景题、计分规则、16 种类型速查表 + 详细解读、输出模板、快速参考。遵循 mbti-test skill 的文件结构和风格模式。

**Tech Stack:** Markdown only, no code dependencies.

**Design Spec:** `docs/superpowers/specs/2026-04-10-behavior-test-design.md`

### Task 2: 编写维度 1-6 测试题（18 题）

**Files:**
- Modify: `skills/behavior-test/SKILL.md`（在 `## 12 维度行为测试` 下追加）

- [ ] **Step 1: 编写执行模式组（维度 1-3）共 9 道题**

**维度一：行动速度**（A = 闪现侠 ⚡，B = 慢半拍 🐢）

每题格式：

```
**[题号]. [具体生活场景]：**
- A) [快速行动的行为选项]
- B) [慢速犹豫的行为选项]
```

设计 3 道题覆盖不同场景（网购、出行、日常决策）。选项只描述**实际行为**，不涉及"你倾向于"或"你更喜欢"等主观措辞。

**维度二：社交引擎**（A = 人形扩音器 🔊，B = 隐形人 🔇）

设计 3 道题覆盖不同社交场景（聚会、线上社交、主动破冰 vs 被动等待）。

**维度三：冒险指数**（A = 赌狗本狗 🎲，B = 安全堡垒 🛡️）

设计 3 道题覆盖不同冒险场景（餐饮、旅行、尝试新事物）。

每维度题目末尾加计分说明行：
```
> 计分：A = [A端标签]，B = [B端标签]
```

- [ ] **Step 2: 编写社交风格组（维度 4-6）共 9 道题**

**维度四：混乱容忍**（A = 强迫症晚期 📋，B = 混沌大师 🌀）
- 3 道题覆盖：桌面/房间整理、数字文件管理、日常物品归位

**维度五：信息深度**（A = 考据狂魔 🔍，B = 刷屏达人 📱）
- 3 道题覆盖：购物决策前的信息搜集、看剧/看书的深度、面对新话题时的行为

**维度六：压力反应**（A = 暴走模式 💥，B = 躺平仙人 🛋️）
- 3 道题覆盖：考试/DDL 前、与人冲突时、遇到倒霉事的反应

### Task 6: 输出报告模板 + 快速参考 + 常见错误 + 总结

**Files:**
- Modify: `skills/behavior-test/SKILL.md`（在详细解读之后追加）

- [ ] **Step 1: 编写输出报告模板**

按设计文档中的模板格式，包含：
- 标题区（类型名称 + emoji + 一句话）
- 四维总览（元维度评分 + 星级）
- 12 维明细（维度名 + emoji + 星级评分）
- 极端行为标签（#标签格式）
- 类型解读占位说明（指导使用者填写内容）
- 行为名场面（场景对话示例）
- 搭配建议（最佳搭档 + 容易互掐）

在模板前加使用说明：
- 如何根据 12 维度得分计算元维度极性
- 如何根据元维度组合查表确定类型编号
- 如何填入各维度星级评分（3:0 → ★★★，2:1 → ★★☆）

- [ ] **Step 2: 编写快速参考表**

| 需要什么 | 去哪找 |
|----------|--------|
| 给某人做行为测试 | 使用"12 维度行为测试"部分，逐维度出题 |
| 快速查看某个行为类型 | 查看"16 种类型速查表" |
| 深入了解某个类型 | 查看对应类型的详细解读 |
| 生成测试报告 | 使用"输出报告模板" |
| 只知道部分维度想推断整体 | 不可以——必须完成全部 12 维度的测试 |

- [ ] **Step 3: 编写常见错误表 + 总结**

常见错误（6-8 条）：

| 错误做法 | 正确做法 |
|----------|----------|
| 把结果当成科学测评 | 本测试是趣味行为观察，无科学依据 |
| 用结果给人贴标签 | 每个人都是独特的，这只是行为倾向参考 |
| 认为某个类型"更好" | 没有优劣之分，每种行为模式都有适用场景 |
| 测试一次就定终身 | 行为模式会随环境、经历变化 |
| 忽略偏好强度只看类型名 | ★★★ 和 ★★☆ 的行为强度差异很大 |
| 在严肃场景中引用测试结果 | 请勿用于招聘、考核、婚恋决策 |

总结（3-5 句）：
- 行为测试是认识自己日常模式的起点，不是终点
- 重点关注极端维度的行为模式
- 善用搭配建议理解不同行为风格的互动
- 明确标注为趣味测试，无科学依据

